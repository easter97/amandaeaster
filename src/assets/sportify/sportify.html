Sportify is a concept I am working on that allows Spotify users to generate custom workout playlists based on the type of exercise they are performing.

I'm someone that is deeply inspired by music and it often directly influences my mood and motivation. When it came to exercise, I found that the workout playlists generated by Spotify were either unspecific to my taste or didn't pick the right songs for the type of energy I needed.

The Idea

Research shows that the right music can boost workout performance and even encourage you to exercise for longer. The music that is best for your workout usually aligns with your target heart rate by matching the BPM, or beats per minute. Here's some thresholds I used for BPM by types of workouts:

  • Yoga, pilates and low-intensity activities: 60 to 90 BPM
  • CrossFit and HIIT: 140 to 180-plus BPM
  • Zumba and dance: 130 to 170 BPM
  • Steady-state cardio, e.g jogging: 120 to 140 BPM
  • Weightlifting: 130 to 150 BPM
  • Warming up: 100 to 140 BPM
  • Cooling down: 60 to 90 BPM

Interface & UI

I build the Sportify application inside of my existing Spotify Comparison app to leverage the services I already built to query the Spotify API. The Sportify interface allows users to select the type of workout they would like to build and what playlists they would like to sample from. I also provided a playlist options tab that allows users to specify the length in minutes or number of tracks they would like to have in the playlist.

Right now, if you don't have enough tracks that meet the BPM criteria in your selected playlists, we just build a playlist with whatever qualifying tracks we have. I'd like to use the Spotify API to add some recommended tracks here to bulk up the generated playlist if we are under the user provided length.

The Math

For the treadmill and treadmill ladder workouts, I used a formula to determine what the target BPM should be for those types of workouts.

Formula to Calculate Steps per Minute:

You can use this formula to estimate the cadence: Cadence (SPM)=Speed (in feet per minute) / Stride Length (in feet))

1 mile per hour (mph) = 88 feet per minute (fpm).

Example Calculation

Speed 3 mph:

3 mph = 264 feet per minute.

Assuming an average stride length of 2.2 feet: SPM = 264 / 2.2 ≈ 120 steps per minute You’d want songs around 120 BPM.

Speed 6 mph:

6 mph = 528 feet per minute.

Assuming a running stride length of 2.5 feet: SPM = 528/2.5 ≈ 211 steps per minute You’d want songs closer to 180-210 BPM for faster running. Matching BPM to Speed:

Speed (mph) Approximate SPM Target BPM
3 mph ~120 SPM 120 BPM
4 mph ~135 SPM 130-140 BPM
5 mph ~150 SPM 140-160 BPM
6 mph ~180 SPM 180 BPM

Playlist Strategy: Start with songs around 120 BPM for a 3 mph pace. Gradually increase BPM as the treadmill speed increases, matching the pace to songs with higher tempos. For running at 6 mph, look for songs around 180 BPM.

Challenges

Unfortunately, Spotify made changes to their developer API in November 2024 that deprecated a lot of endpoints that made Sportify run. I had such a fun time creating this project, so I hope that Spotify will integrate these features back into its developer API to allow applications like Sportify to function again.

Much like my Spotify Comparison application, a fully front-end implementation isn't quite as elegant as retrieving the data from a microservices layer, but we work with the tools we are given. If I was deploying this application on any larger scale, it would definitely benefit from a back end layer to the project.
